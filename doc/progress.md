# 项目进展文档

## 2026-04-12 任务：修复登录/注册页面移动端显示不全问题

### 任务描述
用户反馈登录页面在手机上无法显示底部内容。

### 当前进展
1. **问题分析**：
    - 检查了 `Login.tsx` 和 `Register.tsx` 的布局。
    - 发现使用了 `min-h-[100dvh]` 和 `h-full` 的组合，且父容器有 `overflow-hidden`。
    - 核心问题：`.safe-pb` 在 `index.css` 中定义为 `padding-bottom: env(safe-area-inset-bottom)`。在某些手机浏览器上此值为 0 或被 tailwind 的 `pb-20` 覆盖（取最后定义的），导致底部没有足够的内边距。
    - 另外，容器级 `overflow-hidden` 结合不精确的高度设置在手机端容易截断内容。

2. **已完成修复**：
    - **整体页面结构**：将 `Login` 和 `Register` 的容器统一为 `h-[100dvh]` (匹配 `Layout.tsx`)，移除不确定的 `min-h` 和 `h-full` 冲突。
    - **滚动与安全区域**：在内部滚动容器中添加了 `overscroll-y-contain`，并在内容最底部添加了一个专门的 `div` 占位符 (`h-32 safe-pb`)，确保链接能被滚动到视野内。
    - **个人详情页 (Profile) 修复**：解决了个人信息页顶部的“统计卡片”被截断的问题。原因为父容器 `overflow-hidden` 裁剪了处于 `-bottom-10` 位置的卡片。通过移除父级裁剪并为背景图层建立独立的裁剪容器，完美修复了此处的显示异常。
    - **分析报告页 (AnalysisResult) 优化**：修复了“推荐治疗方案”等模块左侧边框与底部填充视觉不一致的问题。
    - **健康日历 (Calendar) 崩溃修复**：解决了由于缺失 `parseISO` 模块导入导致的页面无法访问（白屏或代码报错）的问题。补充了 `date-fns` 的必要导入，确保历史记录日期解析逻辑正常运行。
    - **全局样式改进**：在 `index.css` 中为 `safe-pb` 和 `safe-pt` 添加了默认值 `0px`，并增加了对 Markdown 容器最后一个子元素边距的自动清除规则，保证了布局的严谨性。
